**Cursor** ist ein KI-first Code-Editor, der auf der Codebasis von VS Code aufbaut. Er integriert KI-Funktionen direkt in den Entwicklungsworkflow und gilt als einer der Wegbereiter des sogenannten Vibe Coding — eines Programmierstils, bei dem Entwickler primär in natürlicher Sprache beschreiben, was der Code tun soll, statt jede Zeile selbst zu schreiben.

## Funktionsweise

Cursor nutzt verschiedene KI-Modelle (Claude, GPT-4o, eigene Modelle) für unterschiedliche Aufgaben. Der Editor indexiert das gesamte Projekt und stellt den Modellen automatisch relevanten Kontext bereit.

### Kernfunktionen

- **Tab-Completion**: Kontextbewusste Code-Vervollständigung, die mehrzeilige Änderungen vorschlägt und mit Tab übernommen wird
- **Inline Chat (Cmd+K)**: KI-gestützte Codebearbeitung direkt im Editor
- **Chat-Panel**: Konversationsbasierte Entwicklung mit Zugriff auf die gesamte Codebasis
- **Agent-Modus**: Autonome mehrstufige Aufgabenbearbeitung mit Tool-Nutzung (Terminal, Dateien, Browser)
- **Codebase-Indexierung**: Semantische Suche über das gesamte Projekt
- **Rules for AI**: Projektspezifische Anweisungen via `.cursorrules`-Datei

## Vibe Coding

Der Begriff Vibe Coding wurde 2025 populär und beschreibt einen Entwicklungsansatz, bei dem KI-Tools wie Cursor den Großteil des Codes generieren. Der Entwickler gibt die Richtung vor, beschreibt Anforderungen in natürlicher Sprache und überprüft die Ergebnisse. Cursor hat diesen Trend wesentlich mitgeprägt.

## Einschränkungen

- Proprietär — kein Open-Source-Code
- Basiert auf VS Code, bietet aber keine vollständige Extension-Kompatibilität
- Premium-Anfragen im Free-Tier stark limitiert
- Datenschutz: Code wird zur Verarbeitung an externe Modelle gesendet (Privacy Mode verfügbar)
- Agent-Modus kann bei komplexen Aufgaben unerwartete Änderungen vornehmen
