Das **Model Context Protocol (MCP)** ist ein offenes Protokoll, das Anthropic im November 2024 veroeffentlicht hat. Es standardisiert die Kommunikation zwischen KI-Anwendungen und externen Datenquellen, Tools und Diensten. MCP wird haeufig als das USB-C der KI bezeichnet — ein universeller Anschluss, der verschiedene Systeme ueber eine einheitliche Schnittstelle verbindet.

## Problem und Motivation

Ohne MCP muss jede KI-Anwendung fuer jede externe Datenquelle eine eigene Integration bauen. Bei M Anwendungen und N Datenquellen entstehen so M x N individuelle Verbindungen. MCP reduziert dies auf ein gemeinsames Protokoll: Jede Anwendung implementiert einen MCP-Client, jede Datenquelle einen MCP-Server — und alle sind untereinander kompatibel.

## Architektur

Das Protokoll definiert drei Rollen:

- **MCP Host**: Die KI-Anwendung, die MCP-Server nutzt (z. B. Claude Desktop, Cursor, eine eigene App)
- **MCP Client**: Die Protokollschicht im Host, die mit Servern kommuniziert
- **MCP Server**: Ein leichtgewichtiger Dienst, der spezifische Faehigkeiten bereitstellt (z. B. Dateizugriff, Datenbankabfragen, API-Aufrufe)

### Faehigkeiten eines MCP-Servers

| Faehigkeit | Beschreibung |
|---|---|
| Tools | Aktionen, die das KI-Modell ausfuehren kann (z. B. Datei erstellen, API aufrufen) |
| Resources | Datenquellen, die das Modell lesen kann (z. B. Dateien, Datenbanken) |
| Prompts | Vorgefertigte Prompt-Templates fuer bestimmte Aufgaben |

## Funktionsweise

1. Der Host startet einen oder mehrere MCP-Server (lokal oder remote)
2. Der Client entdeckt die verfuegbaren Faehigkeiten des Servers
3. Das KI-Modell erhaelt eine Liste der verfuegbaren Tools und Resources
4. Bei Bedarf ruft das Modell ein Tool auf — der Client leitet die Anfrage an den Server weiter
5. Der Server fuehrt die Aktion aus und gibt das Ergebnis zurueck

Die Kommunikation erfolgt ueber JSON-RPC 2.0, entweder via stdio (lokal) oder HTTP mit Server-Sent Events (remote).

## Verbreitung und Adoption

Seit der Veroeffentlichung wurde MCP von zahlreichen Anwendungen uebernommen, darunter Claude Desktop, Claude Code, Cursor, Windsurf und weitere. Es existieren MCP-Server fuer hunderte von Diensten — darunter GitHub, Slack, Google Drive, Datenbanken und Dateisysteme. Die Spezifikation ist Open Source und wird aktiv weiterentwickelt.

## Einschraenkungen

- Sicherheitsmodell erfordert sorgfaeltige Konfiguration (Server haben potenziell weitreichende Berechtigungen)
- Noch kein einheitlicher Standard fuer Authentifizierung bei Remote-Servern
- Qualitaet der Server variiert stark — kein zentraler Zertifizierungsprozess
- Relativ junges Protokoll, Spezifikation noch in Entwicklung
